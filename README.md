# AM-stepbystep

Short and sweet step-by-step lab walkthrough of **Azure Migrate**.

## Sample Data

This lab uses the **`Synthetic_40_VMs.zip`** sample file, which contains a pre-built offline snapshot of 40 virtual machines generated by the Azure Migrate Collector. Download this file and have it ready before starting Part 3.

> **Note:** Importing the ZIP file into Azure Migrate can take **up to 45 minutes** to complete.

## Lab Guide

The full lab guide is in **[lab-guide.md](lab-guide.md)**.

It covers:
1. Creating an Azure Migrate project
2. Enabling preview features via the feature-flag URL
3. Downloading and importing the `Synthetic_40_VMs.zip` offline collector ZIP file
4. Reviewing the Action Center for discovery issues
5. Exploring the discovered inventory (servers, databases, web apps)
6. Tagging workloads (including special preview-only `AzM_Environment` and `AzM_MigrationIntent` tags)
7. Creating application groupings (manually and via CSV)
8. Creating an assessment
9. Reviewing assessment reports, migration paths, and cost estimates
10. Generating PPTX reports from assessments *(preview, requires feature-flag URL)*
11. Using the Azure Copilot Migration Agent *(preview, requires standard portal URL)*

## Notes on Preview Features

Several parts of this lab use features that are currently in preview and require the feature-flag URL introduced in Part 2:

- The reserved tags (`AzM_Environment`, `AzM_MigrationIntent`) in Part 6
- The Reports (Preview) feature in Part 10

The Azure Copilot Migration Agent in Part 11 requires switching **back** to the standard portal URL (`https://portal.azure.com`). Instructions for switching are included in the lab guide.
